# OpenList 插件测试指南

## 1. 构建检查

执行：

```bash
npm run build
```

预期结果：

- `vue-tsc` 通过。
- `vite build` 成功完成。
- 如果最终构建成功，依赖库里的 Rollup 注释警告可以忽略。

## 2. 启动开发服务器

执行：

```bash
npm run dev -- --host 127.0.0.1
```

预期结果：

- Vite 启动在 `http://127.0.0.1:5173/`。
- `public/plugin.json` 中的开发入口指向同一个地址。

当前开发入口：

```json
{
  "development": {
    "main": "http://127.0.0.1:5173"
  }
}
```

## 3. 在 ZTools 中加载插件

完整功能必须在 ZTools 插件环境中测试，因为文件选择、上传、下载和配置持久化依赖 `public/preload/services.js`。

步骤：

1. 打开 ZTools。
2. 加载这个插件目录：

   ```text
   C:\Users\宫城楠木\Desktop\test\my-first-plugin
   ```

3. 使用以下任一命令打开插件：

   - `openlist`
   - `OpenList`
   - `网盘`
   - `文件管理`

预期结果：

- OpenList 插件界面正常打开。
- 如果是在普通浏览器中打开，而不是在 ZTools 中打开，页面应显示运行环境提示。

## 4. 登录测试

需要填写：

- OpenList 地址，例如 `http://your-server:5244`
- 用户名
- 密码

步骤：

1. 填写 OpenList 地址、用户名和密码。
2. 点击 `登录`。

预期结果：

- 登录成功。
- Token 自动保存。
- 根目录 `/` 加载成功。
- 表格中显示文件和文件夹。

失败排查：

- 确认 OpenList 地址没有拼写错误。
- 确认当前机器可以访问 OpenList 服务。
- 确认用户名和密码有效。
- 如果 OpenList 部署在反向代理路径下，确认 base URL 中包含正确路径。

## 5. Token 持久化测试

步骤：

1. 成功登录一次。
2. 关闭插件窗口。
3. 再次从 ZTools 打开插件。

预期结果：

- 插件会读取已保存的 Token。
- 不需要再次输入密码即可加载根目录。

替代测试：

1. 将有效 Token 粘贴到 Token 输入框。
2. 填写 OpenList 地址。
3. 点击 `保存 Token`。

预期结果：

- 插件使用粘贴的 Token 加载文件列表。

## 6. 目录浏览测试

步骤：

1. 点击一个文件夹行。
2. 确认插件进入该目录。
3. 点击 `上级`。
4. 点击面包屑路径中的目录。
5. 点击 `刷新`。

预期结果：

- 点击文件夹后，当前路径会更新。
- 点击面包屑可以跳转到对应父级目录。
- 刷新按钮会重新加载当前目录。
- 底部文件数量会更新。

## 7. 上传测试

建议先使用一个较小的本地测试文件，例如 `.txt` 文件。

步骤：

1. 进入目标 OpenList 目录。
2. 点击 `上传`。
3. 选择一个或多个本地文件。
4. 等待成功提示。
5. 确认上传后的文件出现在当前目录中。

预期结果：

- 插件通过 `PUT /api/fs/put` 上传成功。
- 上传完成后，当前目录会自动刷新。

失败排查：

- 确认当前 OpenList 账号有写入权限。
- 确认目标存储支持上传。
- 先用较小的、英文文件名的文件测试。
- 如果部署在反向代理后面，确认代理允许 `PUT` 请求和 `File-Path` 等自定义请求头。

## 8. 下载测试

步骤：

1. 选择一个普通文件行。
2. 点击 `下载`。
3. 选择本地保存路径。
4. 确认文件已保存。
5. 对比下载文件和原文件内容是否一致。

预期结果：

- 插件调用 `/api/fs/get`。
- 插件下载接口返回的 `raw_url`。
- 保存完成后，系统文件管理器会定位到该文件。

失败排查：

- 确认选择的是文件，不是文件夹。
- 确认 `/api/fs/get` 返回了 `raw_url`。
- 如果 `raw_url` 是相对路径，插件应能基于 OpenList 地址自动补全。

## 9. 回归检查清单

每次改动完成前，建议确认：

- `npm run build` 通过。
- ZTools 可以打开 `openlist` 功能。
- 登录后可以加载 `/` 根目录。
- 文件夹导航正常。
- 使用小文件上传成功。
- 可以下载刚上传的文件。
- 重新打开插件后，可以使用已保存的 Token。
